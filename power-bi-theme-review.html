Visual styles

Rendering overrides applied globally to all visuals and pages

On-brand

All visuals (*.*)

"visualStyles" → "*" → "*" applies to every visual on every page
Property Value What it does
background · transparency 0% Visual backgrounds are fully opaque — the page background colour (#FBFBFB) shows correctly behind all charts.
visualHeader · transparency 100% (hidden) The "…" options menu on each visual is invisible by default — cleaner layout. Still accessible on hover in edit mode.
border · show false No border drawn around visuals. Colour (#E3E3E3) and radius (3px) are defined but inactive — can be switched on per visual without editing the theme.
border · color #E3E3E3 Light grey border — pre-configured but hidden. Ready to enable per visual if needed.
border · radius 3px Slightly rounded corners if border is enabled — consistent with BSI's clean aesthetic.
outspacePane · checkboxAndApplyColor #5EAB96 Filter/slicer pane checkboxes and apply buttons use BSI Green — consistent brand accent for interactive controls.
outspacePane · titleSize 8pt Filter pane title text is 8pt — below brand minimum, but typical for compact filter UI. Flag if accessibility is a concern.
outspacePane · headerSize 8pt Filter pane field header text is also 8pt — same note as above.

All pages (page.*)

"visualStyles" → "page" → "*" applies to every report page canvas
Property Value What it does
background · color #FBFBFB Every report page uses BSI Soft White as its canvas background — reduces eye strain, matches brand guideline for digital backgrounds.
background · transparency 0% Page background is fully opaque — Soft White colour is always visible, never see-through.

Meeting prep

Questions and actions for Pablo, data viz team

Priority questions

Priority · Extended palette
How was the extended palette (slots 9–32) constructed? Is there a system behind the colour choices, or were they taken from a default Power BI template? We'd like to understand this before adding them to Figma as data viz variables.
Priority · Font exception
Noto Sans isn't a built-in Power BI font — is Segoe UI the agreed fallback for all Power BI work? Should we formally document this as the approved exception in Frontify?

Actions for Figma design system

Action · Figma variables
Which colour tokens are most useful to have in Figma as design variables — just the brand 8, or the full 32? Should semantic tokens (good/bad/neutral) and divergent scale stops also be captured?
Action · Figma components
Sunder is building out Figma data viz components. Are there chart types, annotation patterns, or layout conventions in your Power BI work we should capture as reusable Figma components or guidelines?
Action · Maintenance & versioning
As brand guidelines evolve, how should updates to the Power BI theme be managed? Should we establish a versioning process so deployed dashboards can be refreshed centrally?
Action · Semantic tokens
The good/bad/neutral colours aren't in the brand palette. Were these chosen from a specific accessibility or UX framework? Context would help us document them properly in Figma and ensure they're used consistently elsewhere.

Clarifications

Clarify · Other tools
Does the team use other data viz tools beyond Power BI — Excel, Tableau, Python/Matplotlib? Should the same theme be adapted for those, and is there any existing work we can build on?
Clarify · Label size
Data labels are set to 9pt — slightly below our 10pt brand minimum. Was this intentional for dense dashboards? Happy to document as an accepted exception if there's a practical reason.
